# Project Directory Structure - SmartFarm

├── main/
│   └── 프로젝트 최종 실행을 위한 메인 코드들이 들어있는 디렉토리

├── feat/
│   └── 새로운 기능 개발용 브랜치 작업 파일을 보관하는 공간
│       작업자별로 폴더를 나눠 관리 예: feat/cw, feat/sb 등

├── bugfix/
│   └── 기능 오류를 수정한 코드들을 커밋
│       해당 브랜치에서 발견된 버그에 대한 패치 작업이 이루어짐

├── refactor/
│   └── 리팩토링 작업을 위한 공간
│       코드 구조 변경이나 개선 사항을 작업

├── test/
│   └── 테스트 코드나 실험용 코드들을 보관

├── docs/
│   └── 문서화 작업을 위한 공간
│       예: API 명세서, 팀 가이드, 사용법 등이 포함
